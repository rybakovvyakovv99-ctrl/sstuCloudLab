# Автотест лабораторной

Тест поднимает не сервисы, а только проверяет уже запущенный стек по HTTP:
прогоняет `POST /etl/run` и сверяет содержимое витрины и отчётов.

## Как запускать

1. Подними стек из корня проекта:

   ```bash
   docker compose up --build
   ```

2. В другом терминале установи зависимость и запусти тест:

   ```bash
   cd lab/autotest
   pip install -r requirements.txt
   python autotest.py
   ```

   Другой адрес сервиса можно задать переменной окружения:

   ```bash
   ETL_URL=http://localhost:8000 python autotest.py
   ```

## Что показывает

- **Базовые проверки (продажи)** проходят сразу — конвейер по продажам готов.
- **Проверки лабы (возвраты)** будут падать, пока не выполнено задание из
  `lab/LAB.md`. Когда все проверки `PASS`, а код возврата `0` — лаба сдана.

Запуск без Docker (вручную): подними оба сервиса через `uvicorn`
(`source-service` на 8001, `etl-service` на 8000, выставив
`SOURCE_URL=http://localhost:8001`), затем запусти `autotest.py`.
